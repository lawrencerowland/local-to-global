WHAT IS A SHEAF MODEL?

A sheaf provides a rigorous mathematical way to combine local data (each work package's view of scope) into a consistent global picture — or prove that no consistent picture exists.

The Project Graph

Model the programme as a directed graph. Each node is a work package. Each edge represents an interface — shared physical space, shared deliverables, or shared resource claims between packages.

G = (V, E)
V = {WP1, WP2, WP3, WP4}
E = {(WP1,WP2), (WP1,WP3), (WP2,WP4), ...}

Interfaces are where contradictions live. Most programmes have no formal model of interfaces.

Local Sections

Each work package carries local data: its own view of scope items, quantities, cost, and responsibility. This is the "local section" over that node.

F(WP2) = {deck_concrete: 2400m³,
          bearings: "WP2 supply",
          drainage: "WP2 install"}

Local sections are what each work package believes to be true about the overlapping zone.

The Restriction Map

For each edge (interface), a restriction map projects each work package's local data down to the shared interface zone.

ρWP1→WP1∩WP2 : F(WP1) → F(WP1∩WP2)
ρWP2→WP1∩WP2 : F(WP2) → F(WP1∩WP2)

The sheaf condition requires that both restriction maps produce the same value on the interface. If not → incoherence.

Gluing: Building Global Scope

If all local sections are mutually consistent on every interface, the sheaf can be glued into a unique global section — the authoritative programme scope.

∃! global s ∈ F(G) such that
ρV→e(s) = se ∀ edges e

If gluing fails, the programme has no coherent scope. The conflict must be resolved before cost commitment.

Why This Works for UK Infra

UK public sector projects typically have 6–12 major work packages with complex interfaces. NEC contract structures create hard boundaries between packages — making scope conflicts expensive once committed. Sheaf models provide a pre-commitment mechanism that fits perfectly into IPA stage gate reviews.

The Digital PMO Angle

BIM models already capture spatial data per package. The sheaf graph can be derived from federated BIM clash detection reports — turning a manual spatial process into an automated scope coherence engine running on existing data.
